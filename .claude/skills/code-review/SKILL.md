# Code Review — WAG System

Проведи ревью кода. Аргумент `$ARGUMENTS` — путь к файлу или директории. Если не указан — проверь все файлы из `git diff HEAD` (незакоммиченные изменения).

## Шаги

1. **Получи список изменений**
   ```bash
   git diff HEAD --name-only
   # или читай $ARGUMENTS напрямую
   ```

2. **Прочитай каждый файл** и проверь по чеклисту ниже

3. **Запусти TypeScript**
   ```bash
   npx tsc --noEmit
   ```

4. **Выведи отчёт** — по каждому файлу: что хорошо, что исправить, критические проблемы

---

## Чеклист

### Безопасность
- [ ] Нет прямого SQL с интерполяцией строк — только Supabase query builder
- [ ] Server Actions проверяют `getUser()` перед записью в БД
- [ ] Нет `SUPABASE_SERVICE_ROLE_KEY` в клиентском коде (`'use client'`)
- [ ] Нет `console.log` с чувствительными данными (user.id, токены)
- [ ] Входные данные от пользователя не передаются напрямую в запросы без `.trim()` / валидации

### Паттерны проекта
- [ ] Данные из БД получаются в Server Components, не в Client Components
- [ ] Мутации — только через Server Actions в `src/lib/actions/`
- [ ] `'use client'` добавлен только там, где нужен state или event handlers
- [ ] После мутации вызывается `revalidatePath()`
- [ ] Используется `createClient()` из `@/lib/supabase/server` в server-контексте

### Качество кода
- [ ] Нет дублирования логики — переиспользуются существующие actions
- [ ] Нет `any` там где можно вывести тип
- [ ] Обработка ошибок: `.error` из Supabase проверяется
- [ ] Нет `useEffect` для получения данных (только для side-effects)
- [ ] Компонент не делает слишком много — разбит на части если > 200 строк

### UI / Mobile
- [ ] Все padding/margin адаптированы для мобиле: `px-4 md:px-6`
- [ ] Длинные строки имеют `truncate` + `min-w-0` на flex-родителе
- [ ] Новые иконки — из `lucide-react`, не SVG inline (если уже есть в библиотеке)
- [ ] Цвета — через CSS-переменные (`var(--green)`, `var(--text-muted)`), не хардкод

### Логирование активности
- [ ] Создание/изменение/удаление сущностей логируется через `writeLog` или `logActivity`
- [ ] В `meta` передаётся достаточно контекста (имена, не только id)

🔴 Критично: —

### src/components/SomeComponent.tsx

⚠️  Предупреждение: inline style вместо CSS-переменной (#22c55e → var(--green))
🔴 Критично: 'use client' используется только из-за console.log — убрать
